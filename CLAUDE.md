## 專案概述

這是一間客製西裝店（GONY）的訂單管理系統。店主 Eddie 用繁體中文溝通、沒有程式背景。系統前端 Vite + React，後端 Vercel Serverless Functions，資料存在 Notion。操作以手機/平板為主，介面採暗色系（深藍底 #0F1923 + 金色 #C9A84C）。

開發溝通請用繁體中文，避免艱深術語，步驟要具體。

## 技術架構

```
Vite + React（前端，HashRouter 路由）
  src/main.jsx          入口 + 導航 + 4 個分頁
  src/pages/
    OrderEntry.jsx      訂單系統（4 步驟：客戶→樣式→量身→定價）
    Dispatch.jsx        派工管理（建立 7 種派工單）
    DispatchTracking.jsx 派工追蹤（狀態、完成、工資）
    QuickExpense.jsx    快速記帳

Vercel Serverless（後端 /api）
  _notion.js          Notion REST 共用工具（DB id、property builder）
  _styles.js          樣式文字格式化 + 工資計算
  create-order.js     建訂單 + 找/建客戶 + 量身 + 自動財務收入
  orders.js           搜尋訂單
  create-dispatch.js  建派工單
  dispatches.js       列出派工單
  update-dispatch.js  更新派工狀態/工資/送回日期
  expense.js          記帳 + 產生本月固定成本

Notion（資料庫）
  環境變數 NOTION_TOKEN（Integration token）
```

## 核心商業邏輯（定價）

售價公式：`售價 =（工本費 + 布料 + 布損）×（1+利潤率）`，進位到百元。

五品項工本費：二件式 32500、三件式 35450、外套單件 24300、褲子單件 7300、襯衫 1400、背心 7300。

預設碼數：二件式 3.2、三件式 4.0、外套 2.5、褲子 1.4、背心 1.0、襯衫 1.7。布料總價 = 單碼價 × 碼數。

布損級距（襯衫不適用）：≤4000→0、≤6000→1000、≤10000→2500、≤16000→4000、≤20000→5000、≤26000→6500、>26000→7000。

利潤率（一般品項，依布料總價）：≤4000→0%、≤10000→10%、≤16000→15%、≤20000→20%、≤26000→25%、>26000→30%。
利潤率（襯衫）：≤800→55%、≤1500→70%、≤2000→75%、≤2500→80%、>2500→85%。

二件式售價拆分：外套 3/4、褲子 1/4（僅顯示用，師傅工資另計）。

## 派工單系統（7 種）

| 派工單 | 指派 | 內容 |
|--------|------|------|
| 📐 打版單 | 經理 | 完整樣式（量身在訂單關聯的量身記錄）|
| 🧵 毛胚製作單 | 駐店師傅/經理 | 樣式（無量身）|
| 👔 外套製作單 | 外套師傅 | 外套樣式 |
| 👖 褲子製作單 | 褲子師傅 | 褲子樣式 |
| 🦺 背心製作單 | 外套/褲子師傅 | 背心樣式 |
| ✂️ 外套修改單 | 外套師傅 | 修改項目 |
| ✂️ 褲子修改單 | 駐店師傅 | 修改項目 |

派工單直接在 Notion 派工單資料庫建記錄（不用網址生成），Notion 頁面本身就是派工單，傳連結給師傅看。師傅不應看到工資/量身以外的成本資訊。

## 製作流程

接訂單 → 訂布 → 打版 → 毛胚 → 試穿 → 開始製作 → 試穿修改 → 完成。

## 財務系統

財務總表記錄所有收支。欄位：類型(收入/支出)、分類(訂單收入/其他收入/固定成本/人事成本/材料成本/其他支出)、帳戶(🏦銀行/💵現金/💳信用卡)、金額、預估金額/實際金額/匯率差異(布料用)、關聯訂單、付款狀態、結算月份。

- 訂單成立 → 自動產生「訂單收入」記錄（權責發生制，認列全額營收）
- 訂金/尾款是現金流，跟營收分開看
- 布料成本有匯率問題：預估成本（報價）vs 實際成本（叫料後），差額記匯率差異
- 每月可一鍵產生固定成本 97649
